# SymbioForge – Optional digital layer (only when needed)

The core rules work with paper. Digital tools are **not required**.

But if a local circle grows beyond 100 people and finds paper too tedious, a digital version *could* be added – **without changing the six rules**.

## Design principles for any digital version

1. **No blockchain required** – A shared spreadsheet or a simple open‑source app (like a community bank) is enough.
2. **No token speculation** – Digital units are just paper vouchers moved to a screen. No trading on exchanges, no yield farming.
3. **Local control** – Each circle runs its own digital ledger. No global database.
4. **Privacy by default** – Transactions are visible only to circle members (or anonymised).
5. **Forkability** – Any circle can export its data and switch to another software tool.

## Possible open‑source tech stack (example)

- **Backend:** Simple REST API (Node.js, Python, Go) – one instance per circle.
- **Frontend:** Basic mobile web app to send/receive units, view balance, vote.
- **Storage:** SQLite or PostgreSQL – the circle decides.
- **No blockchain, no gas fees, no mining.**

## When to consider digital

- You have > 100 active members.
- Paper vouchers become hard to track.
- Members trust the circle enough to use a shared digital tool.
- Someone volunteers to maintain the software (or uses an existing open‑source community currency system like Community Exchange System or Cyclos).

## Relationship to the core rules

The six rules remain unchanged. Digital is just **an implementation detail**. If a digital tool ever violates the core rules (e.g., creates hidden fees, enables central control, removes exit option), members should stop using it and return to paper.

> **Technology serves the circle. Not the other way around.**
