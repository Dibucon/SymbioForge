# Contributing to SymbioForge

**SymbioForge is not a software project with pull requests and code reviews.**  
It is a **living protocol** for local cooperation. You contribute by using it, testing it, and sharing your experience.

## How you can contribute

- **Start a local circle** – Use the 6 core rules, paper vouchers, and the start guide. After a few months, share your log (see `EXPERIMENT_LOG_TEMPLATE.md`).
- **Open an Issue** – Tell us what worked, what failed, what is unclear. Use the issue templates if possible.
- **Translate** the documents into your language. Add a link to your translation in `TRANSLATIONS.md`.
- **Improve clarity** – If a rule or explanation is hard to understand, suggest better wording via Issue or Pull Request.
- **Share your circle’s log** – Place your completed experiment log in the `/circle-logs` folder via Pull Request.

## What we will NOT accept

- Changes to the **6 core rules** (they are minimal by design – if you disagree, please **fork** the repository and create your own version).
- Adding new global rules, exceptions, or complexity.
- Turning SymbioForge into a blockchain, token, or AI‑governed system.

## Questions?

Open a Discussion (if enabled) or an Issue. Be kind, be constructive.

**When in doubt: keep it simple, keep it local, keep it voluntary.**
